React

Created by Evgeniy Grushevskiy

React is an open source JavaScript library for building user interfaces,authored by Facebook and Instagram Unlike other JavaScript libraries that provide a complete application framework, React focuses on creating application views using components that save state and generate UI elements. You can place a single component on a web page or nest component hierarchies to create a complex user interface. React components are often written in JavaScript and JSX , which is a JavaScript extension that is very similar to HTML and includes some functionality to perform tasks, such as registering event handlers for UI elements. The React component implements a render method that returns JSX code that represents the component's user interface.

Declarative: React makes it painless to create interactive UIs. Design simple views for each state in your application, and React will efficiently update and render just the right components when your data changes. Learn one: We don’t make assumptions about the rest of your technology stack, so you can develop new features in React without rewriting existing code. Virtual-DOM: Since it’s not effective to remove and re-render the entire interface every time state data changes, React uses a fast, in-memory and lightweight representation of the DOM called virtual DOM. React basically uses a very optimized algorithm that compares the virtual DOM of the current and the previous state. The difference between them is what actually needs to be changed in the real DOM and React computes the most efficient DOM mutation for you. In React, for each object of the current DOM there is a corresponding VirtualDOM object. VirtualDOM is an object representation of DOM, its lightweight copy. Virtual DOM contains the same properties as DOM, but cannot directly affect what is displayed on the screen. The image below shows the virtual DOM tree and the comparison process.The red circles represent nodes that have changed. These nodes represent user interface elements whose state has been changed. Then the difference between the previous version of the virtual DOM and the current virtual DOM is calculated. Then the entire parent subtree is re-rendered to get an updated user interface. This updated tree is then batch-updated to the actual DOM model.

Component-Based: Build encapsulated components that manage their own state, then compose them to make complex UIs. Since component logic is written in JavaScript instead of templates, you can easily pass rich data through your app and keep state out of the DOM.

Component-Based

  • Function components

  • Class components

The simplest way to define a component is to write a JavaScript function:

						
							function Welcome(props) {
								return 
Hello, {props.name}
;
							}

You can also use an ES6 class to define a component:

						
							class Welcome extends React.Component {
								render() {
									return 
Hello, {this.props.name}
;
								}
							}

The above two components are equivalent from React’s point of view.

Propses are just arguments that are passed to the component. When React meet JSX attributes, it collects them into one object and passes them to the component. This object is called "props" (props). The props parameter, which is passed to the component function, encapsulates the properties of the object. In particular, the name and age property. When rendering, we can create a set of Hello components, but pass in them different data for name and age. And thus, we get a set of the same type of html markup filled with different data.

The state object describes the internal state of the component, it is similar to props except that the state is defined inside the component and is accessible only from the component.If props represents input data that is passed to the component from the outside, then the state stores such objects that are created in the component and completely depend on the component. Also, unlike props, the values in state can be changed. And another important point is that the values from state should be used during rendering. If some object is not used in component rendering, then it makes no sense to save it in state.
