Who is using React

React is a popular technology using by some large companies as Google, Uber, Twitch, Medium, Sendgrid and etc.
Declarative: React makes it painless to create interactive UIs. Design simple views for each state in your application, and React will efficiently update and render just the right components when your data changes. Learn one: We don’t make assumptions about the rest of your technology stack, so you can develop new features in React without rewriting existing code. Component-Based: Build encapsulated components that manage their own state, then compose them to make complex UIs. Since component logic is written in JavaScript instead of templates, you can easily pass rich data through your app and keep state out of the DOM. Virtual-DOM: Since it’s not effective to remove and re-render the entire interface every time state data changes, React uses a fast, in-memory and lightweight representation of the DOM called virtual DOM. React basically uses a very optimized algorithm that compares the virtual DOM of the current and the previous state. The difference between them is what actually needs to be changed in the real DOM and React computes the most efficient DOM mutation for you. In React, for each object of the current DOM there is a corresponding VirtualDOM object. VirtualDOM is an object representation of DOM, its lightweight copy.VDOM contains the same properties as DOM, but cannot directly affect what is displayed on the screen. The image below shows the virtual DOM tree and the comparison process.The red circles represent nodes that have changed. These nodes represent user interface elements whose state has been changed. Then the difference between the previous version of the virtual DOM and the current virtual DOM is calculated. Then the entire parent subtree is re-rendered to get an updated user interface. This updated tree is then batch-updated to the actual DOM model.

Component-Based

  • Function components

The simplest way to define a component is to write a JavaScript function:

						
							function Welcome(props) {
								return 
Hello, {props.name}
;
							}

You can also use an ES6 class to define a component:

						
							class Welcome extends React.Component {
								render() {
								  return 
Hello, {this.props.name}
;
								}
							}

The above two components are equivalent from React’s point of view.
